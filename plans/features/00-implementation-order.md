## Strategic Context

The existing 309 catalog entries will be **archived and regenerated** by the codex SN build pipeline. Feature 02 (DD Path Linking for existing entries) is therefore **dropped**.

**LinkML assessed and rejected for both grammar and catalog schema:**
- **Grammar:** LinkML defines data schemas (classes, attributes, relationships), not compositional string DSLs. The grammar needs ordered segments, template-based assembly, mutual exclusivity rules, and open-vocabulary parsing. The current `specification.yml` + codegen is purpose-built and correct.
- **Catalog entry schema:** The `models.py` Pydantic models (681 lines) have complex custom validators (unit canonicalization with pint, grammar vocabulary consistency), discriminated unions with per-kind validation, provenance chain normalization, and computed properties. LinkML `gen-pydantic` cannot express any of this — we'd generate a skeleton and override everything. The JSON schema contract (Feature 04) exports directly from the Pydantic models via `.model_json_schema()`.
