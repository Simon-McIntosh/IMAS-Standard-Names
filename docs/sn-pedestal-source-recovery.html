§1 — Problem

The provenance-ledger rebuild drove orphans to zero, but 22 live names (0.8% of 2753) landed on source_type='manual' — the conservative "no deterministic anchor" fallback. Investigation shows this label is wrong for most of them: their DD leaves exist, so a real dd source is derivable. They only became anchorless because they were added directly to the catalog, bypassing the DD-extract pipeline.

Graph evidence: 21 of the 22 carry origin='catalog_edit', catalog_commit_sha=a2f8831, model=None, all imported in one batch (2026-07-04) into edge_plasma_physics.yml (+ a couple of other domains). They were never in any ISNC sources: block (not even the 1941-source high-water mark), and carry no surviving graph anchor — so the rebuild correctly refused to invent a path. But the real DD paths are in the graph.

§2 — Key finding: the sources ARE derivable

The 16 toroidal_<species>_velocity_at_pedestal_top names map deterministically to real DD leaves — the φ (toroidal) velocity of each species at the H-mode pedestal top:

This is a real, existing DD leaf — binding it is recovery, not the fabrication the ledger plan's conservative rule forbids.

The remaining ~6 are individual cases needing a per-name DD lookup: toroidal_plasma_momentum_at_pedestal_top, poloidal_magnetic_flux_at_pedestal_top, normalized_poloidal_magnetic_flux_at_pedestal_top, toroidal_diamagnetic_magnetic_flux_at_inside_flux_surface (origin=pipeline), flux_due_to_thermal_fusion, length_of_iron_core_segment, lower_energy. Some may map to a DD leaf; any with genuinely no DD leaf stay an explicit, auditable manual source.

§3 — Root cause

Two layers:

  • Immediate: these names were hand-added to the catalog to satisfy (likely) an expert review, then imported — so they never received a StandardNameSource from the DD-extract → compose pipeline that would have anchored them to summary/local/pedestal/velocity_phi/*.
  • Systemic: the DD-extract evidently does not surface summary/local/pedestal/velocity_phi/<species>/value as extractable StandardNameSource(dd) leaves (else the pipeline would have composed and anchored them). Needs checking: is summary/local/pedestal/velocity_phi/* classified as a quantity node in SN_SOURCE_CATEGORIES, or filtered out (e.g. the per-species STRUCT_ARRAY leaf-invariant)?

Expert-review comments: the lead indicated these names came from a session addressing expert-review comments; the actual comment text was not captured here — worth attaching to this plan (Review nodes / the PR) to confirm intent, but the DD leaves are the authoritative source regardless.
