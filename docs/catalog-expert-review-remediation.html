§1 — Outcome and scope

This plan converts F. Imbeaux’s 7–10 July 2026 review of catalog release candidates into a current-state remediation programme. It does not assume that release-candidate observations remain true: every item is first checked against the live imas-cx graph, then assigned to the repository that owns the durable fix. Planning and decision capture are in scope; implementation is deliberately deferred.

The desired outcome is a catalog whose names preserve the necessary physics and engineering context, whose definitions are concise and accurate, whose domains are coherent, and whose generator cannot recreate the reviewed defect classes. Renames and duplicate consolidation must preserve lineage and source-path relationships rather than merely deleting conflicting entries.

§2 — Condensed expert review

  1. Documentation discipline: definitions are strong, but additional prose is often verbose, method-specific, speculative, or tangential; examples include thermal_plasma_energy and total_power_due_to_ion_cyclotron_heating.
  2. Provenance presentation: DD source paths should link to DD documentation and were absent in one reviewed release candidate.
  3. Grammar semantics: loci such as LCFS and ITB are usable tokens but have no first-class definitions discoverable in the catalog.
  4. Context preservation: generalising a DD path can erase the owning diagnostic component or local coordinate meaning, as for the X-ray crystal reflector’s x2_curvature.
  5. Coordinate ambiguity: “radial” denotes both cylindrical major-radius direction and a flux-surface-normal or flux label.
  6. State semantics: DD ion state may describe charge, vibrational level, or electron configuration; generated names and definitions over-specialised it to charge state.
  7. Name precision: several names omit plasma, wave-absorption, population, enclosure, or integration semantics, including ICRH power, thermal-electron power, bare power density, stored energy, and first-wall volume.
  8. Duplicate concepts: deposited electron power, plasma energy, and plasma current appear in overlapping bare/total/toroidal forms.
  9. Coordinate completeness: plasma-control and camera objects lack consistent cylindrical coordinate triplets; locus definitions are only indirectly visible through quantity documentation.
  10. Physics correctness: the safety-factor sign text differs from the DD convention, and the minimum-q locus should be based on minimum abs(q) if that is the intended DD concept.
  11. Domain taxonomy: lower-hybrid antenna pressure, computational-workflow gyrocenter quantities, and the broad mechanical-diagnostics bucket appear misplaced or reveal weak domain definitions.
  12. Editorial simplicity: definitions such as flux-loop effective area use needlessly complicated polarity language where a short positive/non-negative constraint would suffice.

§4 — Implementation programme

4.1 Freeze evidence and define policy

Create a machine-readable review ledger keyed by canonical name and finding class. Capture live existence, stage, source paths, grammar decomposition, domain, related names, and the reviewer claim. Lock the decisions below before any rename. Done when each of the 18 rows has an owner, current-state snapshot, decision state, and measurable acceptance test.

4.2 Land low-risk editorial and presentation fixes

Repair DD source hyperlinks, simplify flux-loop wording, and remove unsupported or tangential documentation from confirmed examples. Update the documentation prompt and exemplars at the same time so regeneration does not reverse curated edits. Done when renderer integration tests pass, generated prose meets the proportionality rubric, and the reviewed examples pass expert re-read.

4.3 Resolve semantic duplicates and naming families

For deposited power, plasma energy/current, stored energy, wave power, and enclosed volume, compare source paths and quantity scopes before selecting canonical names. Use predecessor/successor/refined-from relationships for every consolidation. Done when no two accepted entries claim the same source and semantics, all inbound links resolve, and family summaries retain consistent physical bases.

4.4 Clarify grammar-level context

Add vocabulary definitions for loci and settle the radial-frame and reflector/local-axis designs. Any vocabulary change must update YAML and generated enums together, pass full catalog parse/compose round trips, preserve qualifier ordering, and keep related quantities grouped. Done when every relevant token has discoverable semantics and ambiguous source paths can be distinguished programmatically.

4.5 Reclassify domains and close inventory gaps

Define domain assignment policy, audit the three challenged domains, and generate DD-backed coordinate coverage matrices for plasma-control and diagnostic objects. Missing names enter the normal imas-codex proposal/review pipeline; they are not authored directly in ISN. Done when every move has a stated rule, coordinate gaps are either filled or explicitly non-applicable, and cross-domain searches return coherent families.

4.6 End-to-end release gate

Regenerate documentation in a controlled sample, validate the catalog, build the site, query the live graph, and compare the review ledger. Add regression checks for source-link presence, ambiguous radial terms, generic-state over-specialisation, semantic duplicates, domain outliers, and verbosity. Done when every ledger item is fixed, intentionally deferred with a named decision, or documented as already fixed with a passing guard.

§6 — Risks and sequencing constraints

  • Do not treat similar wording as proof of duplication; source scope, aggregation, integration domain, and physical mechanism must be compared first.
  • Do not introduce catalog names directly from this repository. ISN defines grammar; imas-codex proposes names; ISNC stores approved catalog entries.
  • Do not resolve “radial” with a bulk string replacement. Cylindrical components and flux-surface-normal transport require different migrations.
  • Do not rewrite documentation without changing the generator policy and regression corpus, or the next regeneration can recreate the defect.
  • Changes to DD-derived semantics may need upstream DD resolution; quarantine rather than rationalise an authoritative inconsistency.
  • Release-site verification must use the rendered deployment artifact in addition to the graph, because source hyperlink failures are presentation/export defects.

§ Decisions

How should names distinguish cylindrical major-radius components from flux-surface-normal or flux-label quantities?

Recommendation: explicit frame-specific semantics, with migration rules derived from source coordinates.

Must component identity such as reflector be encoded in the standard name, or may structured source-context metadata carry it?

Recommendation: hybrid—encode distinctions that change quantity identity; retain diagnostic-instance context as structured provenance.

What is the primary rule for physics-domain assignment when source subsystem, diagnostic method, and measured physics disagree?

Recommendation: measured physics as primary domain, with source subsystem and diagnostic method retained as secondary metadata.

Should the minimum-safety-factor locus minimize signed q or abs(q), and should resolution wait for an upstream DD clarification?

For accepted ambiguous names, should this programme favour immediate canonical renames or documentation-first correction followed by staged lineage migration?

Recommendation: staged lineage migration except where the current name denotes the wrong quantity.
