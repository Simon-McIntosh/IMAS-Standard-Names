# Data Flow

End-to-end lifecycle of an IMAS standard name, from Data Dictionary path to MCP query result.

## Pipeline Overview

```
DD paths ──▶ codex mint ──▶ graph ──▶ publish ──▶ YAML catalog ──▶ human review ──▶ ISN build ──▶ .db dist ──▶ MCP read
```

## Stage Details

### 1. DD Path Analysis (imas-codex)

imas-codex reads the IMAS Data Dictionary to discover paths that need standard names. It uses ISN's grammar context (`get_grammar_context()`) to understand naming rules before generating candidates.

### 2. Name Minting (imas-codex)

Candidates are generated by applying ISN's grammar rules to DD path semantics. The grammar library functions — `compose_standard_name()`, `parse_standard_name()` — ensure candidates are syntactically valid.

### 3. Graph Organization (imas-codex)

Related candidates are organized into a dependency graph for batch review. This helps domain experts review names in logical groups.

### 4. Publish Candidates (imas-codex)

Candidates are published as pull requests to the [imas-standard-names-catalog](https://github.com/iterorganization/imas-standard-names-catalog) repository.

### 5. Human Review (catalog repository)

Domain experts review proposed names for:

- Physics correctness
- Naming quality and clarity
- Completeness of metadata (unit, description, physics_domain)
- Consistency with existing catalog entries

### 6. ISN Catalog Build (this project)

Once YAML files are merged, ISN builds the catalog:

- Ingests YAML source files
- Runs validation checks (grammar, structural, semantic)
- Produces a SQLite `.db` file
- The `.db` is distributed as part of the `imas-standard-names-catalog` package

### 7. MCP Read Tools (this project)

The ISN MCP server loads the `.db` and serves it through read-only tools:

- **Grammar tools**: `get_grammar`, `compose_standard_name`, `parse_standard_name`
- **Query tools**: `search_standard_names`, `list_standard_names`, `fetch_standard_names`, `check_standard_names`
- **Reference tools**: `validate_catalog`, `get_vocabulary`, `get_tokamak_parameters`

## Where Each Project Fits

| Stage | Project | Role |
|-------|---------|------|
| DD analysis, minting, graph, publish | **imas-codex** | Generates candidate names using ISN grammar |
| YAML source files, human review | **imas-standard-names-catalog** | Stores approved names as reviewed YAML |
| Grammar, validation, catalog build, MCP serve | **imas-standard-names** (this project) | Defines grammar, builds catalog, serves read-only |
