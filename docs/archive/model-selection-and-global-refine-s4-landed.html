Root-fix (a) — derived-parent parse gate (imas-codex 93276ebf)

Root cause: the full-name parse gate is validate_name_candidate (workers.py), which round-tripped every claimed name and quarantined on any parse failure before audits ran. The claim query returned neither origin nor child topology, so a derived family parent — a deliberately partial grammar peel — was indistinguishable from a broken standalone name.

  • Gate scoping: origin='derived' parents are now validated structurally (children exist; parent tokens ⊆ some child's token set), with critical findings for orphans and inconsistent peels — the missed-gate signal is preserved, not papered over.
  • Review-context injection: the names and docs review prompts now render a "derived family parent — score as an abstraction" block listing the children and naming the peeled axis, so review models judge partial names informed.
  • Verification (read-only): of 62 quarantined derived parents live (population grew from the sweep-day 48), all 52 parse-related quarantines clear under the fixed gate; the 4 residual quarantines are genuine unit defects correctly retained. 13 new tests + full regression suite green.
