# IMAS Standard Names Catalog — SPA

Vite + React 18 single-page app that renders the IMAS Standard Names
catalog at <https://Simon-McIntosh.github.io/imas-standard-names-catalog/>.

The SPA is a **pure consumer** of `data.json`: it fetches the dataset at
runtime via `fetch('./data.json')` and re-renders. The dataset is
produced separately by the Python module
`imas_standard_names/catalog/dataset.py`, which writes a `data.json`
alongside the built SPA at deploy time.

## Dev quick-start

```bash
cd site
npm install                  # one-off; needs network access to npm registry
npm run dev                  # http://localhost:5173 — serves the dev fixture
npm run build                # outputs to site/dist/
npm run preview              # serves the built bundle at http://localhost:4173
npm run test                 # runs the vitest smoke tests
```

The dev server reads `site/public/data.json` — a hand-rolled 8-name
fixture committed alongside the source so contributors can develop
without a built catalog. The real 328-name dataset is *not* committed.

## What lives where

| Path | Purpose |
|---|---|
| `src/main.jsx` | React mount + global CSS imports |
| `src/App.jsx` | State, routing, data context, layout shell |
| `src/views/Browse.jsx` | Three-pane Browse layout |
| `src/views/Map.jsx` | Lineage map (SVG cluster cards) |
| `src/components/*.jsx` | All UI atoms (headers, filters, detail, etc.) |
| `src/lib/*.js` | Data loader, indexes, URL state, KaTeX/Mermaid helpers |
| `src/lib/grammar.js` | Role metadata only — **no parsing logic** |
| `src/hooks/useTweaks.js` | localStorage-backed UI preferences |
| `src/styles.css` | Copied verbatim from the prototype |
| `public/data.json` | Dev fixture (8 names) |
| `tests/` | Vitest smoke tests |

## URL routing

Hash-only — no history API, no router library.

```
#/<view>/<encoded-name?>?q=<encoded-query>

#/browse                                      → empty browse view
#/browse/poloidal_magnetic_field              → that name selected
#/browse/safety_factor?q=safety               → name + search query
#/map                                         → lineage map
```

## Hidden flags

| Flag | Behaviour |
|---|---|
| `?tweaks=1` | Mount the floating dev tweaks panel (density / theme / accent / grouping). Lazy-loaded; never lands in the default bundle. Persists to localStorage. |

## Versioned deployment (mike-style)

If a `versions.json` file (mike's standard format,
`[{version, title, aliases}]`) is present one directory above
`data.json`, the header renders a version selector that navigates by
replacing the last segment of the URL path. 404 on `versions.json` is
silent.

## Things to NOT do

- Don't import `data.json` statically. The SPA must work with arbitrary
  datasets generated by the Python emitter and copied in at deploy time.
- Don't add a `parseSN` function. The dataset emitter provides
  `name.parse[]` already tokenised; the SPA only renders.
- Don't put catalog data on `window.*`. Use the `DataProvider` context.
- Don't load KaTeX or Mermaid from a CDN. They come from npm and Vite
  bundles them.
- Don't ship the dev tweaks panel in the default bundle — keep the
  `?tweaks=1` gate.

## Bundle layout

Vite splits vendor code per the `manualChunks` rule in `vite.config.js`:

| Chunk | Contents |
|---|---|
| `vendor-react` | react + react-dom |
| `vendor-katex` | katex runtime |
| `vendor-mermaid` | mermaid runtime — **lazy-loaded** on first `<Mermaid/>` mount |

`oklch()` CSS is used throughout. Floor: Chromium ≥ 111, Safari ≥ 15.4,
Firefox ≥ 113.
