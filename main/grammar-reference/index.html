    Overview

    The IMAS Standard Names grammar defines a structured, deterministic naming convention for fusion data variables. Names are composed of segments in a fixed order, with each segment drawing from controlled vocabularies.

    Vocabularies Summary

    Vocabulary Token Count Usage
    components 12 Vector component directions
    subjects 9 Particle species or plasma subjects
    basis 4 Coordinate system bases
    positions 16 Spatial locations or regions
    processes 26 Physical processes or mechanisms

    Segment Order

    The canonical order for constructing standard names:

    component → subject → base → geometry → position → process → basis

    Where:

    • component (optional)
    • subject (optional)
    • base (required)
    • geometry (optional) - template: of_{token}
    • position (optional) - template: at_{token}
    • process (optional) - template: due_to_{token}
    • basis (optional) - template: in_{token}_basis

    Segment Rules

    Detailed rules for each segment, including optionality, templates, and exclusivity constraints:

    Segment Required Template Vocabulary Exclusive With
    component No — components basis
    subject No — subjects —
    base Yes — (dynamic) —
    geometry No of_{token} positions position
    position No at_{token} positions geometry
    process No due_to_{token} processes —
    basis No in_{token}_basis basis component

    Basis Component Mappings

    Each coordinate basis defines which component tokens are valid:

    Basis Description Components
    cylindrical Cylindrical machine coordinates radial, toroidal, vertical
    field_aligned Directions aligned to or derived from the magnetic field parallel, diamagnetic, normal, tangential, binormal
    cartesian Standard Cartesian axes x, y, z
    poloidal Poloidal plane directions radial, vertical

    Exclusivity Rules

    Certain segments cannot appear together in the same standard name:

    • basis and component are mutually exclusive
    • geometry and position are mutually exclusive

    Examples

    Valid Names

    Using the grammar rules above, here are examples of valid standard names:

    Base scalars:

    • electron_temperature — simple subject + base
    • ion_density — subject + base

    Components:

    • radial_component_of_magnetic_field — component + vector
    • toroidal_component_of_plasma_velocity — component + vector

    With position:

    • electron_temperature_at_plasma_boundary — scalar + position
    • radial_component_of_magnetic_field_at_magnetic_axis — component + vector + position

    With process:

    • heat_flux_due_to_conduction — scalar + process
    • particle_flux_due_to_diffusion — scalar + process

    With basis:

    • radial_component_of_magnetic_field_in_cylindrical_basis — component + vector + basis

    Complex:

    • electron_temperature_at_outer_midplane_due_to_electron_cyclotron_heating — subject + base + position + process

    Invalid Names

    Examples that violate the grammar:

    ❌ magnetic_field_radial_component — component must come first
    ❌ at_plasma_boundary_electron_temperature — segments out of order
    ❌ radial_component_in_cylindrical_basis — both component and basis (exclusive)
    ❌ electron_temperature_at_plasma_boundary_of_magnetic_axis — both position and geometry (exclusive)
